# 5.3 运动管理模块设计

## 5.3.1 模块概述
本模块是"轻养派"健康管理助手的核心功能模块，主要为用户提供运动数据同步、运动记录、智能咨询和久坐提醒服务。通过ExerciseRecord类实现手动/语音两种运动录入方式，通过WeRunData类同步微信运动步数，自动计算运动消耗数据，并通过图表视图展示历史趋势。模块数据将同步至HealthReport类，由AgentService的getExerciseAdvice方法生成个性化运动分析和建议。

## 5.3.2 功能结构设计
运动管理模块包含以下核心功能，完全对应ExerciseRecord和WeRunData类的公共方法：

- 微信运动同步：调用WeRunData.syncFromWeRun()方法，获取微信加密步数数据并解密存储
- 手动运动记录：调用ExerciseRecord.manualRecord()方法，用户手动输入运动类型/时长/强度
- 语音运动打卡：调用ExerciseRecord.voiceRecord()方法，语音描述运动内容自动解析
- 运动后智能咨询：调用ExerciseRecord.askAdvice()方法，结合运动数据生成个性化建议
- 久坐提醒：后端定时检测用户步数，触发微信订阅消息推送
- 运动数据管理：查看、修改、删除ExerciseRecord实例
- 运动趋势查询：按日期查询WeRunData集合和步数趋势图表
- 消耗数据分析：自动计算卡路里消耗、运动时长累计
- AI运动咨询：调用AgentService.getExerciseAdvice()方法生成专业建议

## 5.3.3 业务流程设计
本模块包含五个核心业务流程，对应活动图见本仓库UML设计-运动模块：

1. 微信运动步数同步流程：用户授权→调用wx.getWeRunData→后端解密→调用WeRunData.syncFromWeRun()创建/更新记录→前端渲染图表
2. 手动运动记录流程：用户填写表单→前端计算预估卡路里→调用ExerciseRecord.manualRecord()→保存记录→更新列表
3. 语音运动打卡流程：用户录音→语音转文字→调用ExerciseRecord.voiceRecord()→AI解析→保存记录并返回建议
4. 运动后智能咨询流程：用户输入运动描述→查询历史数据→调用ExerciseRecord.askAdvice()→路由至运动教练Agent→返回个性化建议
5. 久坐提醒触发流程：APScheduler定时任务→查询用户最近1小时步数→判定久坐→调用微信订阅消息接口→推送提醒

## 5.3.4 业务规则
1. **授权规则**
：首次进入运动页需用户授权scope.werun，未授权时显示引导弹窗
2. **步数同步规则**
：每日仅同步一次微信运动数据，重复请求返回缓存结果
3. **卡路里计算规则**
：采用MET公式卡路里 = MET × 体重(kg) × 时长(小时) × 强度系数，不同运动类型对应不同MET值
4. **语音识别规则**
：普通话识别准确率≥85%，识别失败时提示用户手动输入
5. **久坐判定规则**
：最近1小时步数<50且用户开启提醒时触发推送，每日09:00-21:00时段生效
6. **数据同步规则**
：每日23:59自动同步当日ExerciseRecord和WeRunData数据至HealthReport模块
7. **缓存规则**
：微信运动数据缓存24小时，避免频繁调用解密接口

## 5.3.5 接口设计


| 接口名称       | 请求方式 | 接口地址                                  | 对应类方法                         | 功能描述                                                     |
| -------------- | -------- | ----------------------------------------- | ---------------------------------- | ------------------------------------------------------------ |
| 微信运动同步   | POST     | `/api/exercise/werun`                     | `WeRunData.syncFromWeRun()`        | 接收微信加密步数数据，解密后存储并返回趋势                   |
| 手动运动记录   | POST     | `/api/exercise/manual-record`             | `ExerciseRecord.manualRecord()`    | 接收手动填写的运动数据，计算卡路里后保存                     |
| 语音运动打卡   | POST     | `/api/exercise/voice-record`              | `ExerciseRecord.voiceRecord()`     | 接收语音文件，转文字+AI解析后创建记录                        |
| 运动后咨询     | POST     | `/api/agent/exercise-advice`              | `ExerciseRecord.askAdvice()`       | 接收运动描述，路由至运动教练Agent生成建议                    |
| 获取运动记录   | GET      | `/api/exercise/records?date=YYYY-MM-DD`   | -                                  | 获取指定日期的所有`ExerciseRecord`和累计数据                 |
| 获取步数趋势   | GET      | `/api/exercise/werun/trend?days=7`        | -                                  | 获取近N日`WeRunData`步数趋势数据                             |
| 删除运动记录   | DELETE   | `/api/exercise/record/:id`                | -                                  | 删除指定的`ExerciseRecord`实例                               |
| 久坐提醒设置   | PUT      | `/api/exercise/sedentary-setting`         | -                                  | 更新用户久坐提醒开关状态                                     |

# 5.3.7 出错处理设计


| 错误场景                          | 错误码 | 提示信息                         | 处理方式                                   |
| --------------------------------- | ------ | -------------------------------- | ------------------------------------------ |
| 微信运动未授权                    | 40301  | 请先授权微信运动数据             | 跳转授权引导页                             |
| `wx.getWeRunData` 调用失败        | 50001  | 获取步数失败，请重试             | 提示用户检查网络后重试                     |
| 微信数据解密失败                  | 50002  | 数据解析失败，请重新登录         | 清除`session_key`，引导重新登录            |
| 运动类型参数错误                  | 40001  | 不支持的运动类型                 | 提示用户选择有效类型                       |
| 时长超出范围                      | 40002  | 时长需在0-120分钟之间            | 前端校验+后端二次校验                      |
| `AgentService` 调用超时           | 50003  | AI服务繁忙，请稍后重试           | 降级返回通用建议，记录日志                 |
| 语音识别服务不可用                | 50004  | 语音服务暂时不可用               | 提示用户切换手动输入                       |

| 久坐提醒推送失败                  | 50005  | 消息推送失败                     | 记录失败日志，下次重试                     |


# 5.3.8 验收要求
本模块开发完成后，应满足以下验收标准：
1. 微信运动同步：授权成功率≥90%，解密准确率100%，从授权到展示步数总耗时≤3秒
2. 手动记录功能：表单校验准确，卡路里计算与标准公式误差≤15%，保存成功率100%
3. 语音打卡功能：普通话识别准确率≥85%，从录音结束到显示结果总耗时≤3秒
4. 智能咨询功能：建议包含具体数值（如"补充300ml水"），响应时间≤3秒，路由准确率≥90%
5. 久坐提醒功能：触发准确（不误报/不漏报），通知送达率≥95%，推送延迟≤10分钟
6. 数据查询功能：能正确显示任意历史日期的运动记录和步数趋势，无数据时显示"暂无记录"
7. 异常处理：所有异常场景都有明确、友好的提示信息，系统不会崩溃
8. 界面交互：所有按钮点击响应正常，页面跳转正确，折线图渲染流畅无卡顿
