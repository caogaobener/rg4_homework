

# 饮食模块设计说明书

------

## 5.2 饮食管理模块设计

### 5.2.1 模块概述

本模块是"轻养派"健康管理助手的核心功能模块，主要为用户提供饮食记录、营养分析和饮食建议服务。通过`FoodRecord`类实现三种饮食录入方式，自动计算每日营养摄入数据，并通过日历视图展示历史记录。模块数据将同步至`HealthReport`类，由`AgentService`的`getDietAdvice`方法生成个性化饮食分析和建议。

### 5.2.2 功能结构设计

饮食管理模块包含以下核心功能，完全对应`FoodRecord`类的三个公共方法：

- 拍照识食：调用`createByPhoto()`方法，上传食物图片由AI自动识别
- 语音记录：调用`createByVoice()`方法，语音描述饮食内容自动解析
- 手动添加：调用`manualAdd()`方法，用户手动输入食物信息
- 饮食记录管理：查看、修改、删除`FoodRecord`实例
- 饮食日历查询：按日期查询`FoodRecord`集合和累计营养数据
- 营养数据分析：自动计算热量、蛋白质、碳水、脂肪摄入量
- AI饮食咨询：调用`AgentService.analyzeFood()`和`getDietAdvice()`方法

### 5.2.3 业务流程设计

本模块包含四个核心业务流程，对应活动图如下：

1. 拍照识食全流程：用户上传食物图片 → 调用`AgentService.analyzeFood()`识别 → 调用`FoodRecord.createByPhoto()`创建记录
2. 语音饮食记录流程：用户上传语音文件 → 语音转文字 → 调用`FoodRecord.createByVoice()`创建记录
3. 饮食日历查询流程：用户选择日期 → 查询该日期所有`FoodRecord`实例 → 计算累计营养数据
4. 饮食数据同步健康日报流程：每日定时获取当日`FoodRecord`集合 → 调用`AgentService.getDietAdvice()`分析 → 生成`HealthReport`实例

### 5.2.4 界面设计

本模块包含两个核心页面：

1. **饮食记录首页**：展示今日营养累计数据、今日`FoodRecord`列表，提供拍照、语音、文字三种录入入口
2. **饮食日历页**：展示日历组件和对应日期的`FoodRecord`列表，按早餐/午餐/晚餐/加餐分组显示

### 5.2.5 业务规则

1. 图片校验规则：仅支持JPG/PNG格式，大小不超过5MB
2. 识别结果规则：`AgentService`返回的识别结果允许用户手动修改，修改后的数据优先保存
3. 营养累计规则：每日0点自动重置当日`FoodRecord`累计数据
4. 日历查询规则：支持查询所有历史日期的`FoodRecord`，无记录时显示"暂无记录"
5. 数据同步规则：每日23:59自动同步当日`FoodRecord`数据至`HealthReport`模块

### 5.2.7 数据结构设计（**100%匹配类图**）

**FoodRecord实体类**

```java
public class FoodRecord {
    // 私有字段
    private String recordId;      // 记录ID
    private String userId;        // 用户ID
    private String foodName;      // 食物名称
    private Integer calories;     // 热量(大卡)
    private Float protein;        // 蛋白质(g)
    private Float carbs;          // 碳水化合物(g)
    private Float fat;            // 脂肪(g)
    private DateTime mealTime;    // 用餐时间
    private String imageUrl;      // 食物图片URL
    
    // 公共方法
    public FoodRecord createByPhoto() {}
    public FoodRecord createByVoice() {}
    public FoodRecord manualAdd() {}
}
```

### 5.2.8 出错处理设计

| 错误场景                             | 错误码 | 提示信息              | 处理方式                   |
| :----------------------------------- | :----- | :-------------------- | :------------------------- |
| 图片格式错误                         | 40001  | 仅支持JPG/PNG格式图片 | 提示用户重新上传           |
| 图片大小超过限制                     | 40002  | 图片大小不能超过5MB   | 提示用户压缩图片后重新上传 |
| `AgentService.analyzeFood()`调用失败 | 50001  | 识别失败，请重试      | 提示用户重新上传或手动添加 |
| 语音识别失败                         | 50002  | 语音识别失败，请重试  | 提示用户重新录音或手动输入 |
| 日期格式错误                         | 40003  | 日期格式不正确        | 提示用户使用YYYY-MM-DD格式 |
| `FoodRecord`不存在                   | 40401  | 饮食记录不存在        | 提示用户记录已被删除       |

### 5.2.9 验收要求

本模块开发完成后，应满足以下验收标准：

1. 拍照识食功能：常见食物识别准确率≥70%，从上传图片到显示结果的总响应时间≤5 秒
2. 语音记录功能：标准普通话识别准确率≥85%，从录音结束到显示结果的总响应时间≤3 秒
3. 日历查询功能：能正确显示任意历史日期的饮食记录和累计营养数据，无数据时显示 "暂无记录"
4. 数据同步功能：每日 23:59 能自动同步当日饮食数据至健康日报模块，生成对应的饮食分析内容
5. 异常处理：所有异常场景都有明确、友好的提示信息，系统不会崩溃
6. 界面交互：所有按钮点击响应正常，页面跳转正确，底部导航栏固定不滚动
